Grammerly Bot Docs

Everything you need to invite, configure, and get the most out of Grammerly — the Discord bot that corrects spelling so you don't have to.

Overview

Grammerly is a lightweight Discord bot that monitors a designated channel and automatically replies when it detects spelling mistakes. It uses a statistical spell-checker to catch common errors and posts corrections back into the chat — one correction per line, prefixed with an asterisk.

The bot tracks how many times each user has been corrected, supports a customizable word whitelist, and gives server admins full control over whether bot responses are visible only to the user or to the whole channel.

ℹ
Grammerly only monitors one designated channel per deployment. It does not respond to edited messages, and it always ignores other bots.

Quick Start

  1. Invite Grammerly to your server
    Click the Invite Bot button in the navigation and authorize Grammerly with the required permissions for your server.
  2. Designate a channel to monitor
    Grammerly watches a single channel configured via the TARGET_CHANNEL_ID environment variable. Set this to the channel ID of the channel you want corrected. Right-click any channel in Discord and choose Copy Channel ID (you may need Developer Mode enabled in Discord settings).
  3. Configure response visibility (optional)
    By default, correction responses are visible to everyone. Use /configure-responses private to make bot responses only visible to the user who was corrected.
  4. Add words to the whitelist (optional)
    If Grammerly is flagging slang, names, or server-specific vocabulary, use /whitelist-word to add them to the ignore list. Comma-separate multiple words: /whitelist-word gng, bruh, copium.
  5. You're done
    Grammerly is now actively monitoring and will reply to any message containing spelling errors in the designated channel.

Required Permissions

Grammerly requires the following Discord permissions to function correctly. Make sure all four are granted when you authorize the bot.

Read Messages
Allows the bot to see messages in the designated channel.
Send Messages
Allows the bot to post correction replies.
Read Message History
Required for the bot to function in the channel context.
Use Application Commands
Allows slash commands to appear and respond in your server.
⚠
Message Content Intent must be enabled. Go to the Discord Developer Portal → your bot → Bot settings, and toggle on Message Content Intent under Privileged Gateway Intents.

Commands Slash Commands

✓
All commands are Discord slash commands. Type / in any channel to see the available commands listed by Discord's autocomplete.
/latency Anyone

Shows the bot's current latency in milliseconds — the round-trip time between the bot and Discord's API. Useful for checking if the bot is responsive.

Example Response
Grammerly replied: Latency: 42ms
/uptime Anyone

Shows how long Grammerly has been online since its last startup. Useful for verifying the bot is running continuously.

Example Response
Grammerly replied: Uptime: 3 days, 4 hours, 12 minutes
/corrections Anyone

Shows how many times a specific user has been corrected by Grammerly. Correction counts are tracked persistently in a local SQLite database and survive restarts.

Parameters
<user> — required
Example
/corrections user:@Alex
Grammerly replied: Alex has been corrected 17 times.
/whitelist-word Manage Messages

Adds one or more words or phrases to the permanent ignore list. Whitelisted words are never flagged as misspellings, even if the spell-checker would normally correct them. Accepts comma-separated input for batch additions.

Parameters
<word_or_phrase> — required
ℹ
Requires the Manage Messages permission or Administrator. This command always responds privately (ephemeral), regardless of your server's response mode setting.
Examples
/whitelist-word word_or_phrase:copium
/whitelist-word word_or_phrase:gng, bruh, omg, fr
/whitelist-word word_or_phrase:ServerName
/whitelist-list Anyone

Displays all words and phrases currently on the whitelist. If there are more than 25 entries, the response includes Previous and Next pagination buttons so you can browse the full list. The paginator automatically times out after 120 seconds of inactivity.

Example Response
Grammerly replied:
Whitelisted Words:
- bruh
- copium
- gng
- fr
/configure-responses Manage Server

Sets whether Grammerly's correction messages are visible to everyone in the channel (public) or only to the person who was corrected (private). This setting is applied per-server and persists across restarts.

Parameters
<mode> — private | public
ℹ
Requires the Manage Server permission or Administrator. This command always responds privately regardless of the setting you're changing.
Examples
/configure-responses mode:private
/configure-responses mode:public
/help Anyone

Displays a summary of all available commands directly in Discord. Handy for a quick reference without leaving the app.

Environment Variables

Grammerly is configured via environment variables set in a .env file in the bot's root directory. Copy .env.example to .env and fill in the values below.

Variable Required Default Description
DISCORD_TOKEN Yes — Your bot token from the Discord Developer Portal. Never share this.
TARGET_CHANNEL_ID No 1503321842751115314 The ID of the channel Grammerly should monitor for spelling mistakes. Right-click a channel in Discord → Copy Channel ID.
⚠
Never commit your .env file to version control. The .gitignore already excludes it, but double-check before pushing.

Spell-Check Tuning

Advanced spell-check settings can be adjusted in config/settings.py. These control how aggressively Grammerly flags misspellings.

Setting Default Description
SPELL_DISTANCE 1 The edit distance passed to the spell-checker. A value of 1 means words with one insertion, deletion, or substitution from a known word are flagged. Increasing this catches more errors but may produce more false positives.
MIN_WORD_LENGTH 3 Words shorter than this character count are ignored entirely. This prevents the bot from flagging common abbreviations like "ok", "lol", "gg". Increase if you want to catch shorter misspellings.

Response Modes

Grammerly supports two response visibility modes, configurable per server using /configure-responses:

Mode Who Sees It Best For
public (default) Everyone in the channel Servers that want corrections to be a visible, shared experience. Creates public accountability.
private Only the corrected user Servers that want corrections to feel helpful rather than embarrassing. The correction is between the bot and the user only.

Whitelist System

The whitelist allows you to permanently exclude specific words or phrases from being flagged. This is especially useful for:

  • Server-specific slang, nicknames, or terminology
  • Proper nouns that the spell-checker doesn't recognize
  • Internet slang and abbreviations (e.g. gng, bruh, copium)
  • Game names, brand names, or community-specific vocabulary

Whitelist entries are stored in data/ignore_words.txt and persist across restarts. When a phrase is whitelisted, all individual words in that phrase are also excluded when that phrase appears in a message.

✓
Batch add words in a single command by separating them with commas: /whitelist-word gng, bruh, omg, fr, lmao. The bot will confirm how many were added and how many already existed.

How It Works

When a message is posted in the monitored channel, Grammerly runs it through the following pipeline:

  1. Guard checks
    The bot ignores messages from other bots (including itself), messages that aren't in the designated channel, system messages (like join notifications), and empty messages.
  2. Word extraction
    The message text is parsed into individual words, stripping punctuation and casing to lowercase for consistent comparison.
  3. Filtering
    Words shorter than MIN_WORD_LENGTH characters are removed. Whitelisted words and words covered by whitelisted phrases are excluded. Duplicate words within the same message are deduplicated (only flagged once).
  4. Spell-check
    The remaining candidate words are passed to the pyspellchecker library. It uses a word frequency dictionary to identify likely misspellings and suggests the most probable correction for each.
  5. Reply & track
    If corrections were found, the bot replies to the message with each correction on its own line, prefixed with *. The user's correction count is then incremented in the database.
ℹ
Grammerly does not respond to edited messages. Only the original posted message is evaluated. This is by design to prevent correction loops.

FAQ

Why isn't Grammerly correcting messages in my channel?

The most common cause is that TARGET_CHANNEL_ID is set to a different channel than the one you're sending messages in. Double-check the channel ID in your .env file. Also ensure Message Content Intent is enabled in the Developer Portal.

Why is Grammerly flagging a word that's spelled correctly?

The spell-checker uses a frequency-based dictionary and may not recognize slang, proper nouns, or uncommon words. Use /whitelist-word to permanently add any word that should never be flagged.

Are correction counts reset when the bot restarts?

No. Correction counts are stored in a persistent SQLite database at data/corrections.db. They survive restarts, updates, and redeployments as long as the data/ directory is preserved.

Can I monitor multiple channels?

Currently, Grammerly monitors one channel per deployment as set by TARGET_CHANNEL_ID. Multi-channel support is not available out of the box. To monitor a different channel, update TARGET_CHANNEL_ID and restart the bot.

Does Grammerly store message content?

No. Grammerly only stores user IDs and correction counts. It does not log, store, or transmit any message content. See the Privacy Policy for full details.

Can I replace the spell-checker with a custom one?

Yes. Swap out services/spellcheck.py with any implementation that exposes the find_corrections(content: str) → list[tuple[str, str]] interface and the bot's listener will use it automatically.
